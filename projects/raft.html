Raft: dragging FORTH into the modern age

2021 - 2022 (ongoing)

About a year ago, I started learning about stack-oriented concatenative programming languages. That is, languages whose primary method of data manipulation revolves around interacting with a single stack. As I did more research on the topic, my interest grew, and I started taking an interest in FORTH. FORTH is one such stack-based programming language, originally developed in the early 1970s by Chuck Moore. It continues to be used in specialized applications to this day, more than 50 years after its conception.

As such, Raft is the current working name for my most recent project: an implementation of a FORTH-compatible interpreted programming language, designed to combine some of the most convenient features of more modern programming languages with FORTH's simplicity. I began this project after becoming inspired by Alexey Kutepov (Tsoding)'s Porth project. Details of the project can be found in the GitLab repo here: nolanpatrick/raft.

Project Goals

  1. FORTH-compatible*
    • Some programs written in FORTH should run natively in Raft with little-to-no modification.
    • I don't plan on implementing the entire FORTH instruction set, but enough that it is usable as a language.
  2. Approachable
    • Syntax should (optionally) use familiar keywords that most beginner-level programmers would recognize.
    • Program format mimics other popular languages like C, but retains the stack-based style of FORTH.
  3. Easy to learn
    • Familiar syntax combined with some modern amenities and a standard code layout should make it easy to get started, even for someone with little exposure to similar languages.
  4. Fun!
    • Don't leave it to me to tell you that programming doesn't have to be boring. It should be fun, or at the very least, interesting!
